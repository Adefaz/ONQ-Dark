Why is this feature here?

Brightspace updates D2L regularly. The updates may change the structure of the web page, result in broken extension functions, and affect your daily usage.

It's possible that the broken parts will take a few days to be fixed because I sometimes have academic work to do. Before the fix, functions that affect usage will be temporarily disabled by communicating with the server.

What is being sent to the server?

Current extension version (e.g. 1.0.0) and browser name (e.g. Chrome / Firefox). These are not monitored or stored at the server-end. No other information is sent.

Does this slow down page load speed?

No. The extension uses asynchronous request to fetch data. The request and response packets are usually less than several KB in size.

How to disable this feature?

Simply turn off the switch, then the extension will not communicate with the server anymore.
